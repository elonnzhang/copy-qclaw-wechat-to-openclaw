  执行方式：
  
   直接运行（需要 ts-node）
  `npx ts-node openclaw-wechat.ts`

   或指定自定义 openclaw config 路径
   
  `OPENCLAW_CONFIG=~/.config/openclaw/config.json npx ts-node openclaw-wechat.ts`

  脚本做了什么（6步）：

  ┌──────┬────────────────────────────────────────────────────────────────────────────┐
  │ 步骤 │                                    操作                                    │
  ├──────┼────────────────────────────────────────────────────────────────────────────┤
  │ 1    │ 读取 ~/.qclaw/openclaw.json 提取 token & wsUrl                             │
  ├──────┼────────────────────────────────────────────────────────────────────────────┤
  │ 2    │ 读取你的 openclaw config（默认 ~/.openclaw/config.json）                   │
  ├──────┼────────────────────────────────────────────────────────────────────────────┤
  │ 3    │ 注入 channels.wechat-access（含 token 和固定 wsUrl）                       │
  ├──────┼────────────────────────────────────────────────────────────────────────────┤
  │ 4    │ 启用插件 wechat-access & openclaw-qqbot，写入 allow / load.paths / entries │
  ├──────┼────────────────────────────────────────────────────────────────────────────┤
  │ 5    │ 追加 binding（route → clawbot via wechat-access），写回文件                │
  ├──────┼────────────────────────────────────────────────────────────────────────────┤
  │ 6    │ osascript 退出 QClaw，等 2s，执行 openclaw gateway restart                 │
  └──────┴────────────────────────────────────────────────────────────────────────────┘

  ▎ 如果你的 openclaw config 路径不是 ~/.openclaw/openclaw.json，用环境变量 OPENCLAW_CONFIG 覆盖即可。
